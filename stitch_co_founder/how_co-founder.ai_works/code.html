01. INPUT

You Share the Vision

Simply describe your feature, bug fix, or architecture change in plain English. Link a Linear ticket, a GitHub issue, or just type it into the chat. Co-Founder analyzes your codebase to understand context.

> User: "Create a new API endpoint for user profile updates that validates email uniqueness."
02. EXECUTION

I Write & Run Code

The Architect plans the file changes. The Builder writes the actual implementation. Unlike autocomplete, I create multiple files, update dependencies, and run migrations in the sandbox.

03. CORRECTION

I Fix My Own Bugs

If the code fails a test or throws a runtime error, The Critic catches it. I read the stack trace, understand the failure, and rewrite the code until it passes. You never see the broken versions.

04. DELIVERY

You Keep the Keys

Once tests pass and you approve the plan, I open a Pull Request. You review the code just like you would with a human colleague. Merge it, and it's live.

Protocol FAQ

Do you own the code created?
No. You retain 100% intellectual property rights to all code generated by Co-Founder.ai within your environment. It is your repository, your stack, your IP.
Is my codebase used to train other models?
Absolutely not. Your code is processed in ephemeral instances and is never stored permanently on our training servers. We operate under a strict zero-retention policy for training data.
How do I export the code?
Since Co-Founder integrates directly with GitHub and GitLab, "exporting" is as simple as merging a Pull Request. You can also download diff patches directly from the dashboard.

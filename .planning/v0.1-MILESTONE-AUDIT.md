## Previously-Identified Gaps — All Resolved

### Gap 1: Build Progress Polling — FIXED by Phase 12

**Was:** `useBuildProgress.ts:118` called `GET /api/jobs/{jobId}` returning wrong schema.
**Now:** Calls `GET /api/generation/${jobId}/status` with `GenerationStatusResponse` interface matching backend schema (preview_url, build_version, error_message, debug_id all mapped).

### Gap 2: Strategy Graph Field Mismatch — FIXED by Phase 12

**Was:** Frontend read `data.links` but backend `GraphResponse` returns `edges`.
**Now:** Both strategy pages define `ApiEdge` interface with `from`/`to` fields, read `data.edges`, and map to `GraphLink[]` with `source`/`target` for ForceGraph2D.

### Issue 1: Duplicate Route Hierarchy — FIXED by Phase 12

**Was:** Old `/company/[id]/*` routes functional alongside new `/projects/[id]/*` routes.
**Now:** All 3 company route files are thin redirect components (15-16 lines) calling `redirect(\`/projects/...\`)` with query param passthrough.

### Issue 2: Legacy Strategy/Timeline Pages — FIXED by Phase 12

**Was:** Old `/strategy` page had no project context guard.
**Now:** `/strategy/page.tsx` reads `searchParams.get("project")`, guards `fetchGraph` on `!projectId`, and renders "Go to Projects" prompt when no project selected. `/timeline/page.tsx` same pattern.
